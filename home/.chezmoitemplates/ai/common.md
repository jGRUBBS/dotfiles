# Personal working agreements

- Read the repository's own instruction files before making changes.
- Preserve unrelated user changes and avoid destructive Git commands.
- Inspect the current implementation before proposing or applying a change.
- Keep changes scoped, explain material tradeoffs, and validate in proportion
  to risk.
- Prefer established project commands and dependencies over introducing new
  tooling.
- Never print, commit, or copy credentials, tokens, private keys, or local
  authentication state.
- Run targeted tests and report anything that could not be verified.
